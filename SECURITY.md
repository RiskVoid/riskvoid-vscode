# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in RiskVoid, please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, email us at: **hello@riskvoid.com**

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Resolution timeline**: Based on severity

## Scope

This policy covers:

- The RiskVoid VSCode extension
- Any official RiskVoid services

## Recognition

We appreciate responsible disclosure and will acknowledge reporters (unless you prefer to remain anonymous).
